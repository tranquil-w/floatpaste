# Catppuccin 主题收口设计

## 1. 背景

当前仓库刚完成 Catppuccin 主题切换，但界面仍存在一批明显的视觉不一致问题，主要集中在以下几个方面：

- Picker 窗口真实外轮廓与内部内容圆角语言不一致，导致窗口四角与内部卡片关系别扭。
- Picker 高亮项仍沿用偏暖色强调，数字快捷键 `1~9` 在高亮后对比度不足。
- 深色主题下多处边框过重，界面更像“描线分隔”而不是“表面层级分隔”。
- Manager 内的 Radio、Checkbox、输入框、Textarea、次级按钮、状态入口没有形成统一控件语言。
- 文字层级没有随主题切换完成重构，标题、说明、元信息、操作文案之间缺少明确角色边界。
- “查看全部”等操作入口仍接近文本链接观感，不像可点击按钮。

本次工作不是单点修色，而是对 Catppuccin 主题做一次完整收口，让 Picker 与 Manager 在同一套视觉语言下工作。

## 2. 本轮目标

本轮设计目标如下：

1. 统一 Picker 与 Manager 的主题语义，而不是分别局部修补。
2. 为深色主题建立稳定、克制、可长期使用的主强调色逻辑。
3. 重建控件、卡片、文字层级之间的视觉边界，提升界面完成度。
4. 在不大改信息架构的前提下，让界面从“新主题已接入”升级为“新主题已经成体系”。

## 3. 已确认的设计决策

用户已确认以下设计方向，这些选择视为本次实现的固定约束：

- 改造力度：`完整重塑`
- 深色主题重点色语言：`偏冷主导`
- Picker 轮廓策略：`收敛型圆角`
- Manager 整体气质：`展示与工具之间取平衡`
- 文案与文字层级：`纳入本轮统一收口范围`

## 4. 设计原则

### 4.1 统一优先于局部精修

本轮不追求某个模块单独“做得更华丽”，而是优先建立统一的颜色、表面、控件、文字语义。只要某个视觉元素无法归入统一语义，就说明设计仍未收口。

### 4.2 深色主题以冷色承担主结构强调

深色主题中的选中、高亮、焦点、当前项、可交互提示，统一以 `Lavender / Blue` 系为主。暖色不再承担主要结构强调，只用于收藏、暂停、反馈等语义明确的状态提示。

### 4.3 分层靠表面和明度差，不靠重边框

深色主题中，卡片、面板、输入框、列表项之间的分层应优先通过表面亮度差、轻阴影和细弱边框实现。避免大面积高对比描边。

### 4.4 工具界面优先可读、可扫、可操作

Manager 和 Picker 都是高频工具界面。视觉个性可以保留，但不能压过可扫描性、信息层级和交互提示。

### 4.5 文字层级必须与组件层级同步设计

标题、说明、元信息、状态、操作文本不是简单套颜色，而是对应不同语义角色。文字系统必须和卡片、控件一起设计。

## 5. 改造范围

### 5.1 在范围内

- [src/index.css](c:/repos/floatpaste/src/index.css) 中的主题变量与全局表面 token
- [src/features/picker/PickerShell.tsx](c:/repos/floatpaste/src/features/picker/PickerShell.tsx) 的窗口外观、列表项、高亮态、顶栏、数字快捷键
- [src/features/manager/ManagerShell.tsx](c:/repos/floatpaste/src/features/manager/ManagerShell.tsx) 的三栏视觉层级、设置面板控件、收藏预览、详情区按钮组、文字层级
- [src/shared/ui/Panel.tsx](c:/repos/floatpaste/src/shared/ui/Panel.tsx) 的通用面板基调
- 相关状态徽标、次级按钮、输入框、textarea、radio、checkbox 的统一样式

### 5.2 不在范围内

- 信息架构重排
- 功能交互逻辑改动
- 大规模文案重写
- 新增主题模式或新的主题配置能力
- 与本次主题收口无关的后端、数据库、命令层变更

## 6. 主题 token 设计

本次不继续把 `cp-accent`、`cp-yellow`、`cp-surface0` 直接散落在组件中，而是补充更明确的语义层：

### 6.1 表面语义

- 窗口外壳表面：用于 Picker 外层窗口和 Manager 顶级视觉壳体
- 主面板表面：用于 Panel 主体
- 次级卡片表面：用于列表项、信息卡、设置项卡片
- 控件底色：用于输入框、textarea、选择器、次级按钮
- 控件悬浮底色：用于 hover / focus 前的轻反馈

### 6.2 边框语义

- 弱边框：默认卡片、输入框、列表项分隔
- 强边框：当前项、焦点、强调状态
- 焦点光晕：键盘焦点和输入激活态的软高亮

### 6.3 颜色语义

- 主强调色：冷色，负责选中、激活、焦点和主要交互
- 状态暖色：收藏、暂停、成功提示
- 危险色：删除与错误反馈
- 文本主色：正文、标题
- 文本次色：说明文案
- 文本弱色：元信息、标签、辅助提示

### 6.4 结果

组件层不再直接“猜一个颜色”，而是按语义选 token。这样 Picker 和 Manager 才能统一。

## 7. Picker 设计

### 7.1 外轮廓

Picker 采用“收敛型圆角”策略：

- 外层窗口圆角不继续放大到展示卡片感，而保持更利落的工具窗口气质。
- 内部列表项圆角同步收敛，避免外壳偏方、内容偏圆。
- 真实窗口形状与 React 外层容器圆角保持一致，避免视觉错位。

### 7.2 顶栏

- 顶栏背景与正文区保持同主题但略有层次差。
- 顶栏中的品牌点、标题、次级入口采用冷色系统。
- 成功提示文案改为受控的状态提示，不再用高存在感暖色抢占标题区域。

### 7.3 列表项

- 选中项从当前偏黄高亮改为冷色高亮。
- hover 态保留，但明显弱于选中态。
- 深色边框削弱，更多用背景明度变化和轻环线表达当前项。

### 7.4 数字快捷键

- `kbd` 在默认态和选中态都必须保证数字可读性。
- 选中态下数字键不再与高亮背景撞色。
- 数字键视觉角色独立于列表项背景，避免融成一片。

### 7.5 元信息行

- 类型标签、来源应用、时间信息统一弱化，不再和主文本争夺层级。
- 收藏星标保留状态语义，但不再成为高亮项的主视觉焦点。

## 8. Manager 设计

### 8.1 整体基调

Manager 采用“展示与工具之间取平衡”的路线：

- 保留少量品牌感和层次感，但整体降低装饰性发光、重阴影和高饱和强调块。
- 三栏面板回到更稳定的工具界面关系，不再像三张彼此竞争的展示卡片。

### 8.2 左栏

- 左栏英雄卡片降噪，减少高光、发光和过强体积感。
- 收藏预览卡片与下方导航区域统一到同一表面语言。
- “查看全部”从文本链接感改为明确次级按钮感。

### 8.3 中栏

- 搜索框、筛选控件、历史列表项统一到同一控件系统。
- 列表项当前选中态改用冷色主强调色。
- 类型标签、收藏标识、来源应用、时间元信息重新分层。

### 8.4 右栏

- 详情编辑区域与信息卡片统一表面层次。
- 顶部按钮组重新拉开主次关系。
- 编辑区 textarea 背景调整为更贴近 Macchiato surface 的层次底色。
- 成功提示、删除按钮、收藏按钮使用明确语义色，不与主结构高亮混用。

### 8.5 设置页

- Radio、Checkbox 的未选中态必须融入主题，不再停留在浏览器默认控件染色阶段。
- 设置项卡片、输入框、说明文案都使用统一控件与文字语义。
- “跟随系统 / 浅色 / 深色”等配置项保留卡片式分组，但减少厚重描边。

## 9. 共用控件与容器

### 9.1 Panel

- [src/shared/ui/Panel.tsx](c:/repos/floatpaste/src/shared/ui/Panel.tsx) 负责定义三栏的共同基调。
- 边框、背景透明度、阴影和 hover 反馈需要整体收敛。
- 目标是三栏先像一个系统，再让每栏做局部差异。

### 9.2 输入控件

- 文本输入框、数字输入框、textarea 统一底色、边框、焦点环。
- 深色主题下输入区域不应像单独贴上的“黑块”，而应像同一表面系统里的可编辑层。

### 9.3 选择控件

- Radio 和 Checkbox 未选中态从默认中性灰调整为贴近当前 surface 的弱描边。
- 选中态使用主强调冷色。
- 禁用态与可用态要保持足够区分。

### 9.4 按钮

- 主按钮：用于保存、主要执行动作
- 次按钮：用于查看全部、分页、次级切换、普通操作
- 危险按钮：用于删除
- 状态按钮：用于收藏切换、暂停/恢复等语义明确操作

按钮之间的区别应来自语义，不来自组件作者临时拼色。

## 10. 文字层级设计

本轮明确将文字系统纳入设计，不只处理控件和色块。

### 10.1 文字角色

界面中的文字统一分为以下角色：

- 页面标题
- 区块标题
- 卡片标题
- 正文预览
- 辅助说明
- 元信息
- 操作文案
- 状态反馈

### 10.2 设计目标

- 标题必须一眼可识别为标题，不再和正文或说明混成一层。
- 说明文案需要减轻压迫感，改善密度和呼吸感。
- 元信息必须稳定弱化，但不能牺牲可读性。
- 操作文案必须看起来“可操作”，不能继续像普通说明文字。

### 10.3 重点落点

重点处理以下文本区域：

- 左栏页面说明文案
- 收藏预览区标题、内容、时间
- 历史列表项主文本与元信息行
- 详情区模块标题、字段标题与说明
- 设置页每个配置项的标题、描述和帮助文案
- Picker 顶栏标题、列表正文、元信息行和状态提示

### 10.4 文案边界

本轮会调整文字层级和呈现方式，但不做大规模产品命名重写。只有明显不适合当前界面语义的文案才会顺手微调。

## 11. 具体实现建议

### 11.1 先做 token，再做组件

推荐顺序：

1. 在 [src/index.css](c:/repos/floatpaste/src/index.css) 中整理全局语义 token
2. 调整 [src/shared/ui/Panel.tsx](c:/repos/floatpaste/src/shared/ui/Panel.tsx) 作为共同底板
3. 调整 [src/features/picker/PickerShell.tsx](c:/repos/floatpaste/src/features/picker/PickerShell.tsx)
4. 调整 [src/features/manager/ManagerShell.tsx](c:/repos/floatpaste/src/features/manager/ManagerShell.tsx)
5. 收尾检查所有输入控件、按钮、空状态与状态反馈

### 11.2 样式改动与轻微结构改动的边界

只改样式即可解决的问题：

- 主题变量
- 边框、圆角、阴影、背景层级
- 输入框、textarea、radio、checkbox 的外观
- 列表项选中与 hover
- 大部分按钮的视觉语义
- 文字字号、字重、颜色、间距层级

需要轻微结构调整的问题：

- “查看全部”入口需要升级为更明确的次级按钮容器
- 详情区按钮组需要更清晰的主次排列
- Picker 顶栏与列表区的间距关系可能需要轻微重排

## 12. 验收标准

### 12.1 视觉验收

- Picker 外壳与内部条目圆角关系不再冲突
- Picker 选中项与数字快捷键在深色主题下清晰可读
- 深色主题边框不再刺眼
- Manager 的 radio / checkbox 未选中态符合主题
- 编辑区 textarea 背景与整体主题协调
- “查看全部”一眼可识别为可点击操作
- 标题、说明、元信息、操作文案之间的层级清晰

### 12.2 工程验收

- 至少运行一次 `pnpm build`
- 不引入与本次设计无关的功能改动
- 尽量通过 token 和共用样式减少重复定义

## 13. 风险与取舍

### 13.1 主要风险

- 如果只改局部颜色，不整理 token，后续很快会再次失控。
- 如果 Manager 仍保留过强展示感，而 Picker 走收敛工具路线，两者会继续割裂。
- 如果只修控件，不处理文字层级，界面仍然会显得“主题换了，但内容没融进去”。

### 13.2 本轮取舍

- 优先统一语义，不追求每块区域都做出强视觉记忆点。
- 优先解决长期使用的稳定性，不追求一次性大幅重排布局。
- 优先处理用户已明确指出的问题和其背后的系统性原因，不做无关扩展。

## 14. 结论

本次 Catppuccin 主题收口将采用“完整重塑但保持克制”的路线：

- 深色主题偏冷主导
- Picker 更收敛、更像高频工具窗口
- Manager 在展示感与工具感之间取平衡
- 颜色、控件、卡片、文字层级统一进入同一套主题系统

只有这样，当前的 Catppuccin 主题才算真正落地，而不是停留在换色阶段。
