# FloatPaste 测试版发布流程

本文档用于发布首个 Windows 测试版，默认发布形态为 GitHub `Pre-release` + 便携包。

## 发布定位

- 范围：熟人内测，不做公开传播
- 平台：Windows 10 / Windows 11 x64
- 产物：`FloatPaste-v版本号-windows-x64-portable.zip`
- 签名：默认无代码签名，允许首次运行出现系统警告
- 渠道：GitHub Release，必须勾选 `Pre-release`

## 发布前准备

1. 冻结本轮功能，只允许修复阻塞问题。
2. 统一版本号，至少同步以下文件：
   - `package.json`
   - `src-tauri/tauri.conf.json`
   - `src-tauri/Cargo.toml`
3. 按版本补一份发布说明，可直接基于 `docs/测试版发布说明模板.md` 修改。
4. 确认本轮重点验证项仍然聚焦在：
   - 文本复制自动入库
   - 全局快捷键唤起 Picker
   - 键盘选择与回贴
   - 托盘常驻与暂停监听

## 构建命令

首选一条命令完成：

```bash
pnpm release:portable
```

该脚本会自动执行：

1. `pnpm build`
2. `cargo check --manifest-path src-tauri/Cargo.toml`
3. `pnpm tauri build --no-bundle --ci`
4. 复制 `src-tauri/target/release/floatpaste.exe`
5. 生成测试说明、变更说明和 `SHA256SUMS.txt`
6. 输出到 `tmp/release/`

如需指定版本文案模板：

```bash
powershell -ExecutionPolicy Bypass -File scripts/build-portable.ps1 -ReleaseNotesPath docs/测试版发布说明模板.md
```

## 产物清单

脚本执行成功后，`tmp/release/` 下应至少包含：

- `FloatPaste-v版本号-windows-x64-portable.zip`
- `SHA256SUMS.txt`
- 同名解包目录，供本机回归验证

zip 包内应包含：

- `floatpaste.exe`
- `README-测试版使用说明.md`
- `CHANGELOG-版本号.md`

## 发布前人工验收

正式上传前，必须直接运行解包后的 `floatpaste.exe`，而不是只跑开发模式。

必测场景：

1. 启动后能驻留托盘。
2. 复制一段文本后，Manager 中能看到新记录。
3. 全局快捷键能唤起 Picker。
4. `Up / Down / Enter / Esc / 1..9 / Tab` 在 Picker 中正常工作。
5. 在记事本、浏览器输入框、VS Code 中至少各完成一次回贴。
6. 暂停监听后，复制新文本不会继续入库。
7. 关闭 Manager 后应用仍停留在托盘。

任一场景失败时，本轮不发布。

## GitHub Release 清单

1. 创建 tag：`v版本号`
2. 创建 GitHub Release，标题使用：`FloatPaste v版本号`
3. 勾选 `Pre-release`
4. 上传：
   - `FloatPaste-v版本号-windows-x64-portable.zip`
   - `SHA256SUMS.txt`
5. 发布说明中必须写明：
   - 当前仅支持 Windows 10/11
   - 当前无代码签名，首次运行可能出现系统警告
   - 当前为熟人内测，不建议公开传播
   - 如果目标机器缺少 WebView2 运行时，需要先补装
   - 遇到问题时请附目标应用、复现步骤、日志或录屏

## 反馈收集建议

- 用 GitHub Issue 或固定表单收口，不接受零散聊天记录作为正式结论。
- 问题分类固定为：
  - 安装/启动
  - 剪贴监听
  - Picker 唤起
  - 回贴失败
  - 托盘/设置
  - 数据异常
- 出现阻塞问题后，直接发下一个版本，例如 `0.1.0-beta.2`，不要覆盖原有资产。
